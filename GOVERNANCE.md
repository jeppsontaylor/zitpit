# Governance Model

ZitPit is a **community-led** open-source project. Our model is built on transparency, safety, and the Developer Certificate of Origin (DCO).

## Maintainers

Maintainers have write access and are responsible for project direction, review, and merging.
*   **Core Maintainers**: Responsible for architectural decisions and final security review.
*   **Component Maintainers**: Experts in specific adapters (npm, Git, Lab) or infrastructure.

## Decision Making

Decisions are made via Lazy Consensus in Pull Requests:
1.  Propose a change.
2.  Maintainers review and provide feedback.
3.  If no major objections are raised within 48-72 hours, the change is merged.

## Collaboration

We follow a **Strict Security Review** policy for any path changes in `zitpit-core` or the `Mirage Lab`. At least two maintainers must review and approve security-sensitive code.

We value the **Most Brains** working on this. If you are a consistent contributor, you will be invited to become a maintainer.
