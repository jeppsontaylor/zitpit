# Opening Diagram Prompt Pack

These prompts all target the same ultra-simple paper-safe visual language: a sparse vector line diagram with short labels, a few rounded boxes, and no illustration clutter.

## 1. Split panel line diagram

Create a publication-safe vector diagram on a white background with the title `Simple Reality: Unmediated vs. Mediated Intake` centered across the top. Use two light panels side by side. The left panel title is `Unmediated (Direct Risk)` and the right panel title is `Mediated (ZitPit Safety)`. In the left panel, place two small rounded boxes on the left labeled `Human request` and `Agent request`, both feeding by red arrows into one center box labeled `Download / open code`, which then feeds by a single red arrow into one box labeled `Protected host`; add one short bottom label in red: `Direct path to host`. In the right panel, place the same two small request boxes, both feeding by blue arrows into a box labeled `ZitPit gate` with a tiny subtitle `policy check`; from there, route into a smaller amber box labeled `Quarantine + honeypot` with a tiny subtitle `hold / inspect`; from there, route into a blue box labeled `Protected host`; add one short bottom label in blue: `Held before host execution`. Keep the drawing extremely simple: no icons, no clouds, no extra examples, no long captions inside the figure, no decorative effects beyond very light panel tint.

## 2. Minimal control comparison

Generate a very simple conference-paper vector diagram with only boxes, arrows, and short text. Put the title `Simple Reality: Unmediated vs. Mediated Intake` at the top. The left side should be labeled `Unmediated (Direct Risk)` and show two independent sources, `Human request` and `Agent request`, both pointing to `Download / open code`, which points directly to `Protected host`; make the whole left path red and add the small line `Direct path to host`. The right side should be labeled `Mediated (ZitPit Safety)` and show the same two source boxes feeding into `ZitPit gate` with the subtitle `policy check`, then into `Quarantine + honeypot` with the subtitle `hold / inspect`, then into `Protected host`; make the main controlled path blue and the quarantine hold box amber, and add the small line `Held before host execution`. The style must be flat, sparse, professional, and immediately understandable by engineers at a glance.

## 3. Clean box-and-arrow opener

Design a full-width opener figure for a paper using only a handful of rounded rectangles and straight arrows. Use the exact titles `Unmediated (Direct Risk)` and `Mediated (ZitPit Safety)`. The left half must show separate `Human request` and `Agent request` boxes to make clear that both people and agents can independently trigger risky intake; both connect into `Download / open code`, then directly into `Protected host`; the left half should be red and include the label `Direct path to host`. The right half must show the same separate `Human request` and `Agent request` boxes, but they connect into `ZitPit gate` with `policy check`, then into `Quarantine + honeypot` with `hold / inspect`, then finally into `Protected host`; the right half should be blue with only the quarantine hold box in amber, and include the label `Held before host execution`. Do not use icons, actor drawings, clouds, malware art, gradients, or dense explanatory text.

## 4. IEEE-style simplified intake figure

Create an IEEE-friendly vector figure that feels like a simple line chart or flow diagram rather than an illustration. At the top center write `Simple Reality: Unmediated vs. Mediated Intake`. In the left panel titled `Unmediated (Direct Risk)`, show two small source boxes, `Human request` and `Agent request`, merging into one box `Download / open code`, then one arrow into `Protected host`, with a short red label `Direct path to host`. In the right panel titled `Mediated (ZitPit Safety)`, show the same two source boxes merging into `ZitPit gate` with the subtitle `policy check`, then one arrow into `Quarantine + honeypot` with the subtitle `hold / inspect`, then one arrow into `Protected host`, with a short blue label `Held before host execution`. Keep stroke widths consistent, spacing generous, typography bold and legible, and the entire composition very plain and clean.

## 5. Two-actor safety contrast

Make a highly grokable technical diagram with only two actors, two outcomes, and one control path. Title it `Simple Reality: Unmediated vs. Mediated Intake`. On the left, under `Unmediated (Direct Risk)`, use separate `Human request` and `Agent request` boxes to emphasize independent triggering, both feeding into `Download / open code`, then directly into `Protected host`; use red arrows and add the short phrase `Direct path to host`. On the right, under `Mediated (ZitPit Safety)`, use the same separate request boxes, then route them through `ZitPit gate` with `policy check`, then through `Quarantine + honeypot` with `hold / inspect`, then to `Protected host`; use blue for the controlled path, amber for the quarantine box, and add the short phrase `Held before host execution`. The entire image should be minimal enough to render beautifully in a paper: no icons, no scene composition, no background artwork, and no text beyond the short labels specified.
